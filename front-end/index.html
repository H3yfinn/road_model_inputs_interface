Road input glossary

  • Core CSV fields: Branch Path, Variable, Region, Scale, Units, Per..., and 2022. The scenario is always Current Accounts.
  • Partial rows: Fine to leave some cells blank or omit non-essential columns entirely.
  • Ignored values: Blank or non-existent values are skipped by the loader.
  • Stock: Base-year vehicle count for a branch.
  • Mileage: Annual distance travelled per vehicle.
  • Final On-Road Fuel Economy: Energy used per distance travelled.
  • Passenger Vehicle Saturation: Passenger-road-only saturation shared across passenger road.
  • Vehicle Equivalent Weight: Weight used to convert vehicle types into equivalent vehicle activity.
  • Reconciliation Bound Lower: Minimum allowed road-sector reconciliation adjustment.
  • Reconciliation Bound Upper: Maximum allowed road-sector reconciliation adjustment.
  • Reconciliation Weight: Priority weight used when reconciling road-sector values.
  • Sales Share: New vehicle sales split by drive or fuel.
  • PHEV Electric Driving Share: Shared PHEV road-sector series.
  • Survival Rate: Road-sector age series for remaining vehicle vintages.
  • Vintage Profile Share: Road-sector age distribution used for the stock profile.
  • Blank fields: Export the grey provided base-year value unless you enter an override. Missing optional fields are ignored.

Use the comment section to record your data source and other notes. Projected sales shares and mileage and efficiency adjustments are handled within LEAP. Press Run Road Model to generate the LEAP inputs.

Economy

The economy determines which APEC region package is loaded and which region rows are exported.

Data density:

Data density

Less shows only the core variables: stock, stock share, sales share, efficiency and mileage. Efficiency and mileage are grouped at the drive level (one shared value per drive).

More adds: per-fuel efficiency, per-drive mileage, PHEV electric driving share, gasoline/diesel sales share split, reconciliation bounds and weights, age survival profiles, and transport parameters.

Upload Filled CSV

Loads a filled CSV into the interface. The loader reads each row's values and applies them as overrides — blank cells and missing columns are skipped.

Use this to enter data prepared outside the interface, such as a spreadsheet, another tool or even reconciled values after the model has been run.

Download Filled CSV

Exports the current data as a CSV — including all values visible in the interface, both the provided base values and any overrides you have entered.

Run Road Model

Once you are happy with the data, run the road model. This produces:

  • An interactive results dashboard
  • Output values ready to import into LEAP
  • A reconciled CSV you can upload back into the interface if you want to review or adjust the reconciled values
